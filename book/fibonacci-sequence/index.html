    Fibonacci sequence

    You know you can get there, but you just can't reach there.

    We have used recursion for sum and factorial. Though trivial, these basic problems reveal some interesting thinking pattern. Instead of going from the ground zero, we want to look around and see whether we can find something useful which is either known to us or less challenge to reach. Some people call this backward thinking, but we can call this natural thinking.

    This chapter we will continue to apply recursion thinking to another problem, this time the fibonacci sequence.

    Fibonacci sequence

    Write an algorithm to output the nth fibonacci sequence, 1 1 2 3 5 8 13 â€¦. To come up a new number we need to add the previous two numbers.

    If we have been following the previous chapter, we should be able to come up the following recursion plan quickly.

    fib(1) -> 1
fib(2) -> 1
fib(n) -> fib(n-1) + fib(n-2)

    The code can be as short as the plan.

    function fib(n) {
  if (n <= 1) return 1
  return fib(n-1) + fib(n - 2)
}

    I'm proud of you!

    The implementation is super clean, however we need to recognize that this problem is a bit more complex. Why? Because inside each function instance, it calls another function instance twice!
